# 프로젝트 보고서: 문서 기반 RAG 챗봇

## 1. 프로젝트 개요

본 프로젝트는 한국어 문장을 기반으로 한 기본 챗봇에서 출발하여,
RAG(Retrieval-Augmented Generation) 아키텍처를 적용해
사용자의 질문과 관련된 문서를 검색하고 근거 기반 답변을 생성하는
개인 학습용 챗봇 시스템이다.

이 프로젝트의 핵심은 LLM을 새로 학습시키는 것이 아니라,
내 문서를 잘 검색하고 그 문서를 근거로 정확히 답변하게 만드는 것이다.

## 2. 개발 단계

1. 기본 챗봇 데이터 구성 (`chatbot.txt`, `nextword.txt`)
2. 다음 단어 생성 모델 학습 (서브 프로젝트, `train.py` + Colab)
3. RAG 문서 데이터 구축 (`rag_docs/` — RAG, LangChain, FastAPI, LangGraph, Qwen 문서 5개)
4. 문서 청크 분리 (CHUNK_SIZE=300, CHUNK_OVERLAP=50)
5. 임베딩 생성 및 벡터DB 저장 (sentence-transformers + ChromaDB)
6. 사용자 질문 기반 문서 검색 (TOP_K=3, 유사도 임계값 0.35)
7. LLM 기반 답변 생성 (Claude 우선 → Gemini 폴백)
8. FastAPI REST API 배포 (`POST /chat`)
9. 답변 품질 평가 (질문 30개, 100점 점수표, 오차율 = 100 − 점수)
10. 한계점 및 개선 방향 정리

## 3. 환각 방지 설계

문서 근거 없이 LLM이 아는 지식으로 답변하지 않도록 두 겹의 장치를 두었다.

1. **검색 단계 차단**: 검색된 청크의 유사도가 모두 임계값(0.35) 미만이면
   LLM을 아예 호출하지 않고 "문서에서 확인할 수 없습니다"를 반환한다.
2. **프롬프트 단계 차단**: 시스템 프롬프트에 "제공된 문서 내용만 사용",
   "문서에 없으면 확인할 수 없다고 답한다" 규칙을 명시했다.

## 4. 평가 방법

- 평가 질문 30개 (`chatbot/eval_questions.jsonl`):
  문서 기반 질문 10개, 표현 변형 질문 10개, 복합 질문 5개, 문서에 없는 질문 5개
- 자체 점수표 100점 만점: 관련성 30 + 정확성 30 + 문서 기반성 20 + 자연스러움 10 + 완성도 10
- 답변 오차율은 100점에서 최종 점수를 뺀 값으로 정의했다.
- 답변이 검색 문서에 근거했는지(문서 기반성)를 별도 항목으로 평가했다.
- 추가 지표: 검색 성공률@3, 환각률, 응답속도 P95

## 6. 한계점

1. 문서 규모가 작아(5개) 실제 서비스 수준의 검색 품질을 검증하기 어렵다.
2. 글자 수 기준 청크 분할은 문장 경계를 고려하지 않는다.
3. 단일 유사도 임계값으로 답변 가능 여부를 판단해 경계 질문에서 오판 가능성이 있다.
4. LLM 심사자 채점은 재실행 시 점수 변동이 있을 수 있다.

## 7. 개선 방향

현재 프로젝트는 기본 RAG 구조를 중심으로 구현했다. 추후 개선 방향은 다음과 같다.

1. LangGraph를 활용한 Agentic RAG 적용
2. 질문 재작성 Query Rewriting 적용
3. 검색 결과 관련성 평가 단계 추가
4. Hybrid Search 적용
5. Parent-Child Chunking 적용
6. LangSmith를 통한 실행 과정 추적
7. RAGAS 기반 자동 평가 고도화
8. Qwen/Gemma 기반 로컬 LLM 실험
9. LoRA/QLoRA를 활용한 답변 형식 Fine-Tuning
10. GGUF 변환 후 llama.cpp 추론 최적화
