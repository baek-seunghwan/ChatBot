# Sandbox와 안전한 사용

## Sandbox 환경

현재 MOVB는 Kakao Mobility Sandbox API를 이용합니다. 화면과 API에서 주문 생성,
상태 조회와 콜백 흐름을 시험할 수 있지만 실제 기사 배정이나 실제 결제가 일어나는
운영 서비스가 아닙니다.

샌드박스 주문에는 테스트용 연락처 형식을 사용합니다. 저장소와 문서에 실제 개인정보,
실제 연락처 또는 API 비밀키를 커밋하지 않아야 합니다.

## 사용자 확인

견적 조회는 상태를 변경하지 않지만 주문 생성과 취소는 외부 상태를 바꾸는 작업입니다.
MOVB의 채팅 주문은 견적을 먼저 보여주고 사용자의 확인을 받은 뒤 생성합니다. 정보가
바뀌면 기존 견적의 해시와 현재 정보가 다른지 확인하고 다시 견적을 계산합니다.

## 비밀정보와 접근 제어

서버용 API 키는 환경변수에 보관합니다. 브라우저에는 지도 표시에 필요한 JavaScript
키만 전달하고 Kakao Mobility REST API 키와 Vendor ID는 노출하지 않습니다.

회원 비밀번호는 PBKDF2로 해시하고 세션 쿠키는 HttpOnly로 설정합니다. 관리자 API는
역할을 확인하며 일반 사용자에게 사용자·주문 관리 정보를 제공하지 않습니다.

## 답변의 한계

챗봇은 MOVB 구현 문서와 Sandbox 결과를 근거로 답합니다. 실제 운영 요금, 보상,
법적 제한이나 최신 Kakao Mobility 정책처럼 지식 문서에 없는 내용은 추측하지 않고
공식 안내를 다시 확인하도록 설명해야 합니다.
