# 실도로 길찾기와 예약 ETA

## 자동차·다중 경유지 길찾기

MOVB는 카카오모빌리티 길찾기 API를 우선 사용해 출발지부터 경유지와 목적지까지의
실제 도로 거리와 예상 소요 시간을 계산합니다. 경유지가 없으면 자동차 길찾기를,
경유지가 있으면 다중 경유지 길찾기를 사용합니다. 다중 경유지 길찾기는 경유지를
최대 30개까지 받을 수 있지만 MOVB 배송 주문의 경유지는 배송 API 제약을 따릅니다.

KAKAO_REST_API_KEY가 없거나 길찾기 호출에 실패하면 서비스가 중단되지 않도록
보정 거리로 임시 계산합니다. 이때 답변과 화면에 “카카오 실도로”가 아니라
“보정 거리”라고 명시해 실제 데이터인 것처럼 표시하지 않습니다.

## 미래 운행 정보와 예약 배송

픽업 예약 시간이 있으면 미래 운행 정보 길찾기로 해당 출발 시각의 예상 이동 시간을
조회합니다. 카카오 미래 운행 정보는 YYYYMMDDHHMM 형식의 미래 출발 시간을 사용하고,
경유지는 최대 5개까지 지원합니다. MOVB는 예약 시간과 함께 `futureTrafficApplied`
여부를 반환하므로 챗봇이 미래 교통량 반영 여부를 설명할 수 있습니다.

배송 ETA는 Kakao T 퀵·도보 배송 API의 배송 소요 시간 응답과 길찾기 API의 실도로
예상 시간을 함께 보여줍니다. 두 값은 의미가 다를 수 있으므로 하나를 임의로 정답처럼
합치지 않습니다.

## 인증 키 구분

길찾기와 주소 변환은 카카오 디벨로퍼스 REST API 키를 사용합니다. 퀵·도보 배송
주문, 가격, Step 상태와 Sandbox 상태 변경은 별도로 발급받은 Vendor ID와 배송 API
키를 사용합니다. 서버는 두 인증을 분리하며 REST API 키와 Vendor 키를 브라우저에
전달하지 않습니다.
