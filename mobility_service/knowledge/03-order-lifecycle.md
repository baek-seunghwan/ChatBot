# 견적·주문·상태·취소

## 채팅으로 주문하는 순서

1. 사용자가 출발지, 도착지, 물품과 연락처를 말합니다.
2. 챗봇이 빠진 정보가 있으면 한 번에 정리해 다시 묻습니다.
3. 주소를 좌표로 변환하고 배송 견적을 조회합니다.
4. 챗봇이 견적과 주문 내용을 보여주고 명시적인 확인을 요청합니다.
5. 사용자가 “네”, “진행해줘”처럼 확인해야 주문을 생성합니다.

AI가 주소나 연락처를 해석했더라도 사용자 확인 전에는 주문을 생성하지 않습니다.
견적을 본 뒤 정보가 변경되면 이전 가격으로 주문하지 않고 다시 견적을 조회합니다.

## 주문 상태 조회

주문이 접수된 대화에서는 “상태 확인해줘”라고 요청할 수 있습니다. MOVB는 저장된
주문번호를 이용해 Kakao Mobility Sandbox의 현재 상태를 조회하고 로컬 주문 기록을
동기화합니다.

MOVB는 주문 응답에 포함된 실제 Step ID를 사용해 출발지, 각 경유지, 목적지의
상태를 단건 조회합니다. 각 Step에는 진행 상태, 상태 변경 시각, 예상 종료 시각,
주소 정보가 포함될 수 있습니다. 카카오 응답에 아직 Step ID가 없다면 상태를
지어내지 않고 조회 준비 전이라고 안내합니다.

## Sandbox 생명주기 시연

관리자는 Sandbox 화면에서 배송원 배정 완료, 픽업 완료, 배송 완료, 취소, 강제 종료
상태를 변경할 수 있습니다. 전체 생명주기 시연은 배송원 배정 → 픽업 완료 → 배송
완료 순서로 호출합니다. 이 기능은 Sandbox 전용이며 운영 주문에 사용하는 기능이
아닙니다.

## 주문 취소

접수 전이라면 작성 중인 배송 정보를 초기화할 수 있습니다. 이미 접수된 주문이라면
저장된 주문번호를 기준으로 취소 API를 호출합니다. 취소 가능 여부와 결과는 Sandbox
응답을 기준으로 안내하며, 챗봇이 임의로 성공했다고 말하지 않습니다.

## 중복과 상태 안전성

MOVB는 동일한 partnerOrderId의 주문을 중복 생성하지 않습니다. 같은 콜백이 반복되면
중복으로 기록하고, 이미 진행된 주문 상태를 과거 단계의 늦은 콜백이 되돌리지 못하도록
보호합니다.
