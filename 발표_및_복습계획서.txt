# 발표 준비 + 복습계획서

## 1. 프로젝트 한 줄 요약

카카오모빌리티 퀵 API에 AI 챗봇을 결합해, 채팅으로 배송 접수부터
**서로 모르는 사람끼리의 "퀵 합승"**까지 되는 서비스를 만들었다.

## 2. 만든 것

**① 대화형 배송 접수 (배송 에이전트)**
- 채팅: "판교역에서 정자역으로 서류 퀵 보내줘" → AI가 주소·물품·연락처 추출
  → 좌표 변환(카카오 로컬 API) → 견적 → 사용자가 "네" 확정 시에만 실제 주문
- LangGraph 상태 그래프로 의도분류 → 정보추출 → 검증 → 견적 → 확정 흐름 관리
- LLM: Claude Haiku 4.5 기본, 장애 시 Gemini 2.5 Flash Lite 자동 폴백

**② 퀵 합승 — 핵심 기능** 🚚
- 카카오 퀵의 "경유지 추가 혜택"은 **한 사람** 물건만 묶을 수 있다.
  합승은 **서로 다른 발송인**의 물건을 AI가 방향으로 매칭해 한 차로 묶는다.
- 플랫폼은 주문 2건이 1건으로 줄어 매출이 깎이므로 만들 이유가 없는,
  구조적으로 비어 있는 자리 → "기업이 못 하는 기능"
- 매칭 조건: 진행 방향 차이 ≤ 30° + 묶은 주행거리가 단독 합의 85% 이하
- 요금 분배: 각자 단독 견적에 비례해 나눔 (멀리 가는 사람이 조금 더 냄)

**③ 택시 동승 (카풀) 요금 분배** 🚕
- 채팅: "강남역에서 A는 잠실역, B는 건대입구역 가는데 요금 어떻게 나눠?"
- AI가 **누구를 먼저 내려줘야 총 이동이 최단인지** 모든 방문 순서를 비교해 결정하고,
  요금은 각자 혼자 탔을 때 요금에 비례해 공정하게 분배
- 검증 예시: 총 13.0km, 요금 13,900원 → 건대(B) 먼저 하차,
  분담 B 6,700원 / A 7,200원 — 각자 3천 원 이상, 둘이 합쳐 6,200원 절약
- 카카오는 2019년 카풀 서비스를 택시업계 반발로 접었다 →
  이것도 "기업이 이해관계 때문에 못 하는" 영역
- ※ 택시는 공개 API가 없어 요금은 거리 기반 추정치 (배송 합승과 달리 실주문 없음)

**④ 부가 기능**
- 묶음 배송 비교(따로 vs 묶음 자동 견적 비교)
- 좌하단 플로팅 챗봇 위젯 (AI 채팅 / 로컬 모델 토글), 합승 대기 보드
- 별도 프로젝트: 문서 기반 RAG 챗봇 (ChromaDB+FAISS+BM25 하이브리드 검색)

## 3. 합승 실제 검증 결과 (Sandbox)

| 구분 | 경로 | 요금 |
|---|---|---|
| A 단독 | 강남역 → 대전 유성구 | 100,100원 |
| B 단독 | 잠실역 → 대전 서구 | 102,200원 |
| 따로 합계 | | **202,300원** |
| 합승 1건 | 강남픽업→잠실픽업→서구하차→유성하차 | **118,000원 (42%↓)** |
| 분담금 | A 58,400원 / B 59,600원 | 각자 4만 원 이상 절약 |

전 과정(등록→매칭→확정→실제 Sandbox 주문 생성→상대방 결과 확인)을 검증 완료.

## 4. 발표 데모 시나리오

브라우저 창 2개(하나는 시크릿 모드)로 A/B 두 사용자 역할:
1. A: "합승으로 싸게, 강남역→대전 유성구, 서류..." → 대기 등록 + 자동진행 동의
2. 합승 보드에 A의 대기 건이 뜨는 것 보여주기
3. B: "잠실역에서 대전 서구로 상자 하나 합승으로" → 매칭 + 요금 비교표
4. B "네" → 실제 주문 생성 (주문번호 pool-...)
5. A 창에서 "내 합승 어떻게 됐어?" → 매칭 결과 + 분담금 확인

## 5. 예상 질문 Q&A

- **왜 카카오는 이걸 안 하나요?** → 기술이 아니라 매출 구조 문제. 2건 요금이 1건으로 줄어드는 기능이라 플랫폼이 먼저 만들 이유가 없다.
- **매칭은 어떻게 판단하나요?** → 방위각 차 ≤ 30° + 묶은 총거리 ≤ 단독 합의 85%. 실익이 있을 때만 성립.
- **잘못 주문되는 건 어떻게 막나요?** → 사용자가 명시적으로 확정해야만 주문 실행 + 중복 확정은 결정론적 주문 ID로 차단.
- **어떤 모델을 썼나요?** → Claude Haiku 4.5 (폴백 Gemini). 파인튜닝 없이 프롬프트로 의도분류/정보추출/응답을 나눠 지시.
- **실서비스 가능한가요?** → 아직 Sandbox 단계. 실서비스는 카카오 프로덕션 승인 필요.
