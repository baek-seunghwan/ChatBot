MOVB 택시_퀵모브 (MOVB)
기능 소개 택시 접수하기 퀵 접수하기 이용 내역
서비스 홈
MOVB · 카카오모빌리티 API + 챗봇 AI, LOCAL

새로 추가한 기능은 두 가지입니다

① 퀵 합승 — 모르는 사람끼리 물건을 한 차에 묶어 요금을 나누는 배송 시스템(실제 주문까지 동작). ② 택시 동승 — 같은 원리를 사람에 적용한 하차 순서·요금 분배 계산. 두 기능 모두 접수와 판단을 AI 챗봇이 처리합니다.

기능 01 · 퀵 합승

같은 방향이면, 모르는 사람끼리도 한 차로

서울→대전 퀵 한 건은 10만 원이 넘습니다. 카카오 퀵의 "경유지 추가 혜택"은 한 사람 물건만 묶을 수 있지만, 이 시스템은 서로 다른 발송인의 물건을 AI가 방향으로 매칭해 한 차로 묶습니다. 기사님은 한 번만 달리고, 요금은 나눠 냅니다.

요금이 이렇게 달라집니다 Sandbox 실제 견적으로 검증

지금 방식 — 차량 2대, 요금 2번
202,300원 두 사람 요금 합계
42% 절약
퀵 합승 — 차량 1대, 요금 나눔
118,000원 202,300원
A 분담 강남역 → 대전 유성구 · 서류 58,400원 −41,700원
B 분담 잠실역 → 대전 서구 · 상자 59,600원 −42,600원

분배 규칙: 합승 요금 118,000원을 각자 단독 견적 비율(100,100 : 102,200)로 나눔. 멀리 보내는 사람이 조금 더 내되, 둘 다 혼자보다 훨씬 쌉니다.

합승 시스템은 이렇게 굴러갑니다

채팅 한 줄로 접수

AI가 문장에서 출발지·도착지·물품·연락처를 추출하고, 카카오 로컬 API로 주소를 좌표로 바꾼 뒤, 단독 견적부터 조회해둡니다. 정보가 빠지면 그 항목만 되묻습니다.

"강남역에서 대전 유성구로 서류 하나, 합승으로 싸게 보내줘"

대기 보드에 등록

아직 같은 방향이 없으면 합승 대기 보드에 올라갑니다. 이때 "매칭되면 확인 없이 바로 진행해도 될까요?"라고 물어 사전 동의를 받아둡니다 — 나중에 상대가 나타났을 때 자리를 비워도 합승이 성립되게 하는 장치입니다.

새 요청이 들어올 때마다 매칭 판정

두 조건을 모두 통과해야 합승이 성립합니다: ① 두 배송의 진행 방위각 차이 ≤ 30° (같은 방향인가) ② 한 차로 도는 총 주행거리가 각자 따로 갈 때 합의 85% 이하 (묶어서 실익이 있는가). 반대 방향이거나 우회가 커지면 매칭하지 않습니다.

경유 동선 설계

차량은 픽업을 전부 마친 뒤 하차하는 순서로 돕니다. 픽업들과 하차들 각각을 가까운 곳부터 방문하는 순서로 정렬해 동선을 만듭니다 (강남 픽업 → 잠실 픽업 → 서구 하차 → 유성 하차).

요금 비교표 → 명시적 확정

"따로 202,300원 vs 합승 118,000원, 내 분담금 59,600원"을 보여주고 사용자가 '네'라고 답해야만 주문이 실행됩니다. 같은 제안에 '네'를 두 번 보내도 주문번호가 결정론적으로 생성되어 중복 접수가 차단됩니다.

카카오모빌리티 API로 실주문은 1건

픽업 2곳→하차 2곳을 경유지로 엮은 실제 퀵 주문이 생성되고(pool-412a4789b02c, Sandbox 검증), 두 사람 모두 자기 채팅에서 "내 합승 어떻게 됐어?"로 매칭 결과·분담금·배송 상태를 확인합니다.

기능 02 · 택시 동승

같은 원리를 사람에게 — 하차 순서와 요금 분배

"강남역에서 A는 잠실, B는 건대 가는데 요금 어떻게 나눠?" — AI가 모든 방문 순서를 비교해 총 이동이 가장 짧은 하차 순서를 정하고, 요금은 각자 혼자 탔을 때 요금에 비례해 나눕니다.

검증 예시

B 분담 건대입구 · 먼저 하차 (혼자 타면 9,700원) 6,700원 −3,000원
A 분담 잠실 · 끝까지 탑승 (혼자 타면 10,400원) 7,200원 −3,200원

총 요금 13,900원을 단독 요금 비례로 분배 — 멀리 가는 사람이 더 내되 전원이 이득. 택시는 공개 API가 없어 거리 기반 추정 요금으로 계산합니다 (실호출 없음).
