Qwen은 알리바바가 공개한 오픈소스 LLM 계열입니다.
0.5B부터 70B 이상까지 다양한 크기가 있어 로컬 환경에서도 실행할 수 있습니다.

파인튜닝(fine-tuning)은 이미 학습된 모델을 내 데이터로 추가 학습시키는 과정입니다.
Qwen을 파인튜닝하는 이유는 다음과 같습니다.
첫째, 내 도메인의 말투와 형식에 맞는 답변을 하도록 만들 수 있습니다.
둘째, 프롬프트에 매번 긴 지시문을 넣지 않아도 원하는 행동을 하게 됩니다.
셋째, API 비용 없이 로컬에서 실행되는 나만의 모델을 가질 수 있습니다.

전체 파인튜닝은 모델의 모든 가중치를 다시 학습하므로 GPU 메모리가 많이 필요합니다.
그래서 소규모 환경에서는 LoRA를 주로 사용합니다.
LoRA는 Low-Rank Adaptation의 약자로,
원본 가중치는 고정하고 작은 어댑터 행렬만 학습하는 방식입니다.
학습해야 하는 파라미터가 크게 줄어들어 적은 메모리로도 파인튜닝이 가능합니다.

양자화(quantization)는 모델 가중치의 숫자 정밀도를 낮춰 크기를 줄이는 기술입니다.
예를 들어 16비트 가중치를 4비트로 줄이면 모델 크기와 메모리 사용량이 크게 감소합니다.
양자화를 하는 이유는 큰 모델을 일반 GPU나 노트북에서도 실행하기 위해서입니다.
정밀도가 낮아지면 품질이 약간 떨어질 수 있지만,
4비트 양자화 정도는 실사용에서 체감 차이가 크지 않은 경우가 많습니다.
QLoRA는 양자화된 모델 위에 LoRA를 적용해 더 적은 메모리로 파인튜닝하는 방법입니다.

RAG와 파인튜닝은 목적이 다릅니다.
RAG는 지식을 밖에서 찾아오는 방법이고, 파인튜닝은 행동 방식을 모델 안에 새기는 방법입니다.
최신 정보나 문서 기반 답변이 필요하면 RAG,
말투나 출력 형식을 바꾸고 싶으면 파인튜닝이 적합하며 둘을 함께 쓸 수도 있습니다.
