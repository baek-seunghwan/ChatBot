LangGraph는 상태(state) 기반 그래프로 LLM 워크플로를 만드는 프레임워크입니다.
LangChain 팀이 만들었으며, LangChain 위에서 동작합니다.

LangChain과 LangGraph의 차이는 흐름의 모양입니다.
LangChain의 체인은 단계를 한 방향으로 순서대로 연결합니다.
질문 → 검색 → 답변처럼 직선 흐름에 적합합니다.
LangGraph는 노드와 엣지로 이루어진 그래프를 만들기 때문에
조건 분기, 반복, 되돌아가기 같은 복잡한 흐름을 표현할 수 있습니다.

LangGraph의 핵심 개념은 세 가지입니다.
State는 그래프 전체에서 공유되는 데이터입니다. 예를 들어 질문, 검색 결과, 답변을 담습니다.
Node는 상태를 받아 작업을 수행하고 상태를 갱신하는 함수입니다.
Edge는 어떤 노드에서 어떤 노드로 이동할지 정하는 연결선이며, 조건에 따라 경로를 바꿀 수 있습니다.

LangGraph가 유용한 경우는 다음과 같습니다.
검색 결과가 부족하면 질문을 바꿔 다시 검색하는 self-correction 흐름,
답변 품질을 검사해서 기준 미달이면 재생성하는 흐름,
여러 도구 중 상황에 맞는 것을 골라 쓰는 에이전트 흐름입니다.

단순한 RAG 챗봇은 LangChain 체인이나 직접 구현으로 충분하고,
분기와 반복이 필요해지는 시점에 LangGraph를 도입하는 것이 일반적입니다.
