LangChain은 LLM 애플리케이션을 쉽게 만들 수 있게 도와주는 파이썬 프레임워크입니다.
문서 로딩, 청크 분리, 임베딩, 벡터DB 연동, 프롬프트 구성, LLM 호출 같은
RAG 파이프라인의 각 단계를 부품처럼 조립할 수 있게 해줍니다.

LangChain을 사용하는 이유는 반복되는 연결 코드를 줄이기 위해서입니다.
예를 들어 문서를 나누는 TextSplitter, 벡터DB를 감싸는 VectorStore,
검색을 담당하는 Retriever 같은 표준 인터페이스를 제공합니다.

LangChain의 핵심 개념은 체인(Chain)입니다.
체인은 여러 단계를 순서대로 연결한 것으로,
질문 → 검색 → 프롬프트 구성 → LLM 호출 → 답변이라는
RAG의 흐름을 하나의 체인으로 표현할 수 있습니다.

LangChain 없이도 RAG를 만들 수 있습니다.
직접 청크를 나누고 임베딩 모델과 벡터DB를 호출하는 코드를 작성하면 됩니다.
소규모 학습 프로젝트에서는 직접 구현이 동작 원리를 이해하는 데 더 도움이 되고,
규모가 커지면 LangChain으로 옮기는 것이 유지보수에 유리합니다.

LangSmith는 LangChain 팀이 만든 관측 도구입니다.
체인의 각 단계에서 어떤 프롬프트가 들어가고 어떤 응답이 나왔는지 추적할 수 있어
RAG 파이프라인을 디버깅하거나 평가할 때 사용합니다.
